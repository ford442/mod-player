### Root Cause Summary
The channel-count mismatch was **already fixed in HEAD**. The packer and GPU buffer allocator both derive `numChannels` from `rawChannels + (padTopChannel ? 1 : 0)`, and the shader receives the same `numChannels` via uniforms. The historical "320 cells into 256-cell buffer" log no longer reproduces. The only remaining gap was the absence of a runtime invariant that would catch a future regression.

## Iteration 2 — Amber-vs-Blue Emitter Differentiation — 2026-04-24T16:03:00+02:00

### Status
Complete

### Files changed
- `shaders/patternv0.50.wgsl`

### What I found
- The v0.50 shader already had three emitters and DURA sustain logic, but the top emitter was always blue (via `calculateBlueIntensity`) and the bottom was always amber for expression-only steps.
- The shader constants `NOTE_MAX = 96u`, `NOTE_OFF = 97u`, etc. did not match the packer's `NOTE_MAX = 119` / `NOTE_OFF_MIN = 120`. This was a shader-side bug that would have misclassified notes 97–119.
- The data path uses a `storage` buffer (`cells: array<u32>`) with flat varyings, not an `RG32UI` texture as described in the task brief. The actual bit layout in `packPatternMatrixHighPrecision` is:
  - packedA: `[note:8 | inst:8 | duration:8 | volPacked:8]`
  - packedB: `[effCmd:8 | effVal:8 | durationFlags:7 | reserved:1 | volCmd:8]`
- The packer already pre-computes the expression-only flag in bit 7 of `inst`, so the shader can use `isExpressionOnly` directly without reconstructing it from volCmd/effCmd.

### What changed
1. **Fixed note-range constants** to match packer: `NOTE_MAX = 119u`, `NOTE_OFF_MIN = 120u`.
2. **Replaced dead functions** `isSustaining` and `calculateBlueIntensity` with a new `calculateTopIntensity` that branches on `isNoteOn / isExprOnly / isSustain`.
3. **Reclassified cell types** in the fragment shader:
   - `isNoteOn`: note > 0 && note < 120 && rowOffset == 0
   - `isNoteOff`: note >= 120
   - `isExprOnly`: !isNoteOn && !isNoteOff && isExpressionOnly
   - `isSustain`: note > 0 && note < 120 && duration > 1 && rowOffset > 0 && !isNoteOff
   - `isDead`: everything else
4. **Rerouted top emitter**: blue (`#0080FF`) for note-on, amber (`#FFA500`) at ~0.7 intensity for expression-only, very faint blue (0.1) for sustain, dark for dead/note-off.
5. **Rerouted middle emitter**: pitch-class color at full intensity for note-on, ~45 % fixed intensity for sustain, dark (`#0D0D0D`) for expression-only and dead.
6. **Rerouted bottom emitter**: suppressed for expression-only (amber moved to top); only subtle amber when note-on/sustain rows also carry expression data.
7. **Updated external glow** to use new `isNoteOn / isSustain` booleans.
